Course Projects

  • Java Mini Compiler

    Supervised by: Dr. Swarnendu Biswas
    Compiler Design, Jan'23-Apr-23

    The aim of the project was to create a mini compiler for a subset of Java. The compiler should be able to generate the assembly code, which can then be run on a simulator. The lifecycle of the compiler is as follows:

    • Lexical Analysis: The input code is tokenized and converted into a stream of tokens. The tokens are categorised into different types: keywords, identifiers, operators, literals and separators. This was achieved by creating regular expressions in Flex.
    • Syntax Analysis: Next, the tokens are parsed to check if they follow the grammar of the language using Bison. The corresponding Abstract Syntax Tree (AST) is generated using Dot.
    • Semantic Analysis: We then create the symbol table and check for semantic errors. The symbol table is created manually using C++ vectors and maps.
    • Code Generation: Then, the 3ac code is generated using the AST. The 3ac code is then converted into assembly code while mantaining the registers and memory locations in a separate data structure. This part is completey written in C++. Finally, the assembly code is written to a file and is run using NASM.

  • Evaluating Cache Security

    Supervised by: Dr. Debadatta Mishra
    Computer Architecture, Sep'22-Nov-22

    The aim of this project was to evaluate the security of the cache memory. We highlighted the vulnerabilities faced due to shared cache memory and posibilities of cache-based side-channel attacks. The simulations were done on associative cache memory with random replacement using CacheFX.

    We created eviciton set for the cache using three algorithms: Single Holdout Method(SHM), Group Elimination Method(GEM) and Prime+Prune+Probe(3P) and compared their compute time, accuracy and memory usage.

    Sample Scenario: Suppose a process is running on a system and contains some sensitive data. Just before the data is about to be accessed, the attacker can run a process which will evict the data from the cache. Thus, when the sensitive data is fetched from the memory, it will be loaded into the cache. The attacker can then use the eviction set to identify the sensitive data.
    However, building an eviction set is a costly task and cannot be created in such a short time. But with increasing computational power, these kinds of attacks may be a threat in the future.
